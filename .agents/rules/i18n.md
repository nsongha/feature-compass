# i18n (Internationalization)

## Pattern hiện tại
- Object `I18N` chứa 2 locale: `en` và `vi`
- Truy cập text qua `App.t(key)` — fallback sang `en` nếu key không tồn tại trong locale hiện tại
- Ngôn ngữ lưu trong `App.state.lang`, persist qua localStorage

## Khi thêm text mới — BẮT BUỘC
1. Thêm key vào **CẢ HAI** `I18N.en` và `I18N.vi`
2. Dùng `App.t('keyName')` trong code — KHÔNG hardcode string
3. Key dùng camelCase: `featureIdea`, `evalComplete`, `noEvalIdeas`

## Ví dụ đúng
```javascript
// Trong I18N object
en: { myNewLabel: 'My Label' },
vi: { myNewLabel: 'Nhãn mới' },

// Trong render code
`<div>${this.t('myNewLabel')}</div>`
```

## Ví dụ SAI
```javascript
// ❌ Hardcode string
`<div>My Label</div>`

// ❌ Chỉ thêm EN, quên VI
en: { myNewLabel: 'My Label' },
// vi: thiếu!
```

## Phân loại keys
- **UI labels**: `featureIdea`, `settingsTitle`, `portfolioQuadrant`
- **Actions**: `evalAI`, `deleteBtn`, `exportMd`
- **Toasts/messages**: `evalComplete`, `deleted`, `setApiKey`
- **Verdicts**: prefix `v` cho tên (`vBuildNow`), prefix `vd` cho description (`vdBuildNow`)
- **Dimension hints**: `impactHint`, `fitHint`, `effortHint`, `conflictHint`
- **Context relations**: `supports`, `conflicts`, `constrains`, `overlaps`

## AI Response Language
- Khi `lang === 'vi'`, prompt gửi thêm instruction: "Respond entirely in Vietnamese"
- AI analysis fields (summary, reasoning) sẽ trả về bằng ngôn ngữ tương ứng
